# Boundra v0.1.1

## Release Summary

Boundra의 첫 사용 가능한 public preview입니다. TypeScript 애플리케이션은
`boundra` npm 패키지의 schema-backed runtime을 사용하고, Rust 기반 CLI는
도메인 생성, 계약 코드 생성, 경계 검사와 의존성 그래프를 담당합니다.

## Highlights

- BR-001부터 BR-004까지 client/server/shared/cross-domain 경계를 정적
  분석합니다.
- `create-domain`, `add-dependency`, `generate route/query/mutation`,
  `check-boundaries`, `graph-domains` CLI workflow를 제공합니다.
- 생성 계약은 Zod schema와 `boundra` runtime을 통해 입력과 결과를
  실행 시점에도 검증합니다.
- 진단은 stable error code, file/line context, 수정 suggestion과 JSON
  출력을 제공합니다.
- npm runtime과 Linux, macOS Intel/Apple Silicon, Windows CLI artifact를
  clean-room 및 플랫폼 CI에서 검증합니다.

## Installation

TypeScript runtime과 schema authoring dependency를 설치합니다.

```bash
pnpm add boundra@0.1.1 zod
```

CLI는 이 GitHub Release의 플랫폼별 archive를 내려받아 `PATH`에 추가하거나
Rust toolchain으로 설치할 수 있습니다.

```bash
cargo install --git https://github.com/qtaghdi/boundra --tag v0.1.1 boundra-cli
```

설치 후 다음 명령으로 확인합니다.

```bash
boundra --version
boundra --help
```

## Breaking Changes

- 잘못 게시된 `boundra@0.1.0`은 지원하지 않습니다.
- pre-release 문서와 생성 코드에서 사용하던 `@boundra/runtime` 대신
  `boundra`를 import합니다.

## Known Limitations

- npm 패키지는 TypeScript runtime을 제공하며 네이티브 CLI 바이너리는
  아직 포함하지 않습니다.
- 생성된 Zod schema는 안전한 빈 객체로 시작하므로 실제 domain field를
  작성해야 합니다.
- 빈 프로젝트를 구성하는 `boundra init` 명령은 아직 없습니다.
